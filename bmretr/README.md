# bmretr — Benchmark for first 10 CV/JD extraction

Compares QA-based (`retrieinfor`) vs embedding-based (`retrieinfor1`) extraction against a human-curated gold standard.

## Directory structure

```
bmretr/
├── scripts/           # Runnable scripts
├── gold/              # Human-curated perfect extraction
├── gold_mocked/       # Gold with empty fields filled for fair comparison
├── raw/               # Raw input CSVs (what goes into the retrievers)
├── outputs/           # Retriever outputs
│   ├── qa/            #   retrieinfor output
│   └── emb/           #   retrieinfor1 output
└── results/           # Benchmark results (generated by bench.py)
```

## File contents

### scripts/
| File | What it does |
|------|-------------|
| `create_gold.py` | Creates gold standard from phase1 markdown tables |
| `create_mocked_gold.py` | Fills empty education/major fields in gold so both systems are fairly compared |
| `run.py` | Runs both retrievers on the first 10 CV/JD |
| `bench.py` | Token-overlap (word probability) benchmark vs gold |

### gold/
| File | Content |
|------|---------|
| `cv.csv` | Perfect extraction for 10 CVs — `UserID`, `summary`, `education_level`, `institution`, `major`, `skills`, `location`, `salary_expectation`, `desired_job` |
| `jd.csv` | Perfect extraction for 10 JDs — `JobID`, `job_description`, `responsibilities`, `requirements`, `required_education_level`, `required_major`, `required_skills`, `salary_offer`, `job_location` |

### gold_mocked/ (same schema as gold/)
| File | Content |
|------|---------|
| `cv.csv` | Some education/major fields filled with reasonable defaults where gold had blanks |
| `jd.csv` | Some required_education/major fields filled with defaults from JD text |

### raw/
| File | Content |
|------|---------|
| `cv_first10.csv` | First 10 CVs formatted as model input |
| `jd_first10.csv` | First 10 JDs formatted as model input |
| `cv_raw.csv` | Original raw CV data (source) |
| `jd_raw.csv` | Original raw JD data (source) |

### outputs/qa/ and outputs/emb/
Each contains `cv.csv` and `jd.csv` — the structured extraction output from each retriever.

### results/
| File | Content |
|------|---------|
| `detail.csv` | Per-record, per-field precision/recall/f1 for both systems vs gold |
| `detail_mocked.csv` | Same but against mocked gold |
| `summary.csv` | Macro averages grouped by system/type |
| `summary_mocked.csv` | Macro averages against mocked gold |
| `best.csv` | Per-field winner (QA vs EMB) with precision diff, using mocked gold |
| `wins.csv` | Decisive wins only (large precision diffs) |
| `zero_analysis.csv` | Zero-precision analysis + best method per field (original gold) |

## Workflow

```bash
# 1. Generate gold (one-time)
python bmretr/scripts/create_gold.py

# 2. Generate mocked gold (optional)
python bmretr/scripts/create_mocked_gold.py

# 3. Run both retrievers on first 10
python bmretr/scripts/run.py

# 4. Run benchmark
python bmretr/scripts/bench.py
```

## Metric

Token-overlap (word probability): for each field, lowercased words from prediction ∩ gold divided by prediction size (precision) and gold size (recall). Aggregated to macro F1.
